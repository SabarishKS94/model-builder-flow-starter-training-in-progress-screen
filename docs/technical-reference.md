This document keeps the **longer, platform-level notes** for this repo. It matches the “Salesforce UI starter” style of documentation you may have seen in generic LWC-on-Vite templates. For a **Data 360–oriented** overview, start with the [root README](../README.md).

### Component namespaces

| Folder     | Tag prefix | Use for |
|------------|------------|--------|
| **shell/** | `shell-*`  | App shell only. Not for feature pages. |
| **page/**  | `page-*`   | One folder per URL / route. |
| **ui/**    | `ui-*`     | Reusable building blocks. |
| **data/**  | —          | Plain JS modules, `import ... from 'data/...'`. Not LWC tags. |

Do not add a `lightning/` namespace under `src/modules` for custom components. Icon shims: only `src/build/lightning-icon/shims/`.

**Examples:** `src/modules/page/dashboard/` + route in `routes.config.js` and `ROUTE_COMPONENTS` in `app.js` → `page-dashboard`. `src/modules/ui/card/` → `<ui-card>`.

## Adding pages and UI

1. **Page:** `src/modules/page/<name>/`, then register in `src/routes.config.js` and in `ROUTE_COMPONENTS` in `src/modules/shell/app/app.js`. For routes that share a parent tab, use `navHighlight` instead of a new `navPage`.
2. **UI block:** `src/modules/ui/<name>/`, use as `<ui-<name>>`.
3. **Modals:** Extend `LightningModal` from `lightning/modal` — see `src/modules/ui/demoModal/`. Prefer that over raw SLDS modal markup.

## Routing

- **Config** — `src/routes.config.js`: `{ path, component, title, navPage?, navLabel?, navPath?, navHighlight? }`.
- **Path params** — e.g. `/users/:id`; `getCurrentRoute()` in `src/router.js`.
- **Navigation** — `navigate(path)` from the router; shell re-renders the matching `page-*` component.

## Tech stack and dependencies

- **vite** / **vite-plugin-lwc** — build and LWC
- **lwc** + **@lwc/synthetic-shadow** — platform-like shadow behavior
- **lightning-base-components** — `lightning-*` components
- **@salesforce-ux/design-system** / **design-system-2** — SLDS 1 & 2 (Cosmos) CSS via `src/build/slds-loader.js`
- **public/images/** — static assets (e.g. favicon)

## SLDS 1 and SLDS 2

**SLDS 2** (Cosmos) is default. `initSldsFromStorage()` in `src/index.js` runs before the app mounts. The loader uses `<link rel="stylesheet" data-slds="...">` and `media` toggling. **SLDS 1** loads on demand from `src/build/slds/slds1-url.js`. Icon templates are generated by `prebuild-icons.mjs`.

## Shadow DOM (synthetic)

Default is **synthetic shadow** to align with the Salesforce host. In `vite.config.js`, set `disableSyntheticShadowSupport: true` in the LWC plugin only if you need native shadow for a specific experiment.

| | Synthetic (default) | Native |
|-|------------------------|--------|
| Platform match | Closer to Salesforce | Different |
| `shell-app` shadowRoot | `null` | `ShadowRoot` |

Check: `document.querySelector('shell-app').shadowRoot` → `null` means synthetic.

## Icons

`npm run dev` / `build` runs `prebuild-icons.mjs` and updates `src/build/generated/`. Vite maps `lightning/iconSvgTemplates*` to shims under `src/build/lightning-icon/shims/`.

## Conventions

Prefer **Lightning Base Components**, then **SLDS** utility classes, then styling hooks. This repo may ship **`.cursor/rules`** for SLDS and LWC. Official docs: [SLDS](https://lightningdesignsystem.com), [LWC (Salesforce)](https://developer.salesforce.com/docs/component-library/documentation/en/lwc).

## Deployment and platform

This project is for **local prototyping** and design-system-aligned UI; deploying artifacts to a Salesforce org follows normal product and release processes and is not covered here.
